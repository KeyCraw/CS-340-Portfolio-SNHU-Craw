# CS 340 Portfolio – Client/Server Development

Welcome to my CS 340 portfolio. This repository showcases my work in client/server application development, database systems, and full-stack design. It demonstrates how backend databases, server-side logic, and client-side interfaces work together to create functional, data-driven applications.

---

## Course Overview

CS 340 focuses on client/server development, database systems, and data management. The work in this repository reflects the application of MongoDB, Python, and dashboard frameworks to design and implement a complete client/server system.

Throughout this course, I worked on a project for Grazioso Salvare, an organization that trains rescue dogs. The project required building a database, developing a reusable server-side CRUD module, and creating a client-facing dashboard to interact with the system.

---

## Grazioso Salvare Project Summary

The Grazioso Salvare project involved developing a client/server application that helps identify dogs suitable for rescue training. The system uses data from the Austin Animal Center and provides tools for filtering and analyzing that data.

The application allows users to:
- View animal shelter records in an interactive data table  
- Filter dogs based on rescue categories such as water, wilderness, and disaster tracking  
- Visualize data through charts and graphs  
- View animal locations using a geolocation map  
- Interact with a MongoDB database through a Python-based server module  

In Project One, I developed the database and created a reusable CRUD Python module that serves as the server-side component responsible for handling database operations.

In Project Two, I built a dashboard application that acts as the client, sending requests to the server-side CRUD module and displaying the returned data. This demonstrates the interaction between client and server in a full-stack system.

---

## What I Did Well

One of my strongest contributions in this project was developing a reusable CRUD Python module that clearly separated server-side database logic from the client-side dashboard. This made the system more maintainable, readable, and adaptable.

I also successfully implemented a client/server interaction model, where the dashboard acts as the client and communicates with the MongoDB database through the server-side module. This separation of responsibilities improved organization and made the system easier to expand.

## What I Would Improve

If I were to improve this project, I would enhance the user interface of the client dashboard to make it more visually polished and user-friendly. While the system functions correctly, improving layout and design would make it more professional.

## Interpreting and Implementing User Needs

To interpret the client’s needs, I focused on how Grazioso Salvare selects dogs for rescue training. The client required a system that could efficiently filter and analyze large datasets.

I implemented these needs through:
- Client-side filtering controls in the dashboard  
- Server-side data retrieval using the CRUD module  
- Visualizations to support decision-making

Considering user needs is essential because a system must be both technically correct and usable. This project demonstrates how client/server systems can support real-world decision-making.

## My Approach to Designing Software

When designing software, I begin by understanding the problem and identifying system components. In this project, I focused on separating the system into:

- Database (MongoDB)  
- Server-side logic (CRUD Python module)  
- Client-side interface (dashboard application)  

I also use a step-by-step approach:
- Build and test each component independently  
- Integrate components gradually  
- Validate functionality at each stage  

In future projects, I will continue using this modular client/server approach to build scalable and maintainable systems.
